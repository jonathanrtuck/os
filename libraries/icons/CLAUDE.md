# icons

Named vector icon library. Pure data — no rendering, no color, no theme knowledge. Icons are pre-compiled from Tabler Icons (MIT) SVGs at build time into native path commands.

## Key Files

- `lib.rs` -- `get(name, mimetype)` lookup function, `Icon`/`IconPath`/`Layer` types, `mimetype_category()` helper
- `data.rs` -- **Generated by build.rs** — const path data arrays, lookup tables, fallback. DO NOT EDIT.

## API

```rust
pub fn get(name: &str, mimetype: Option<&str>) -> &'static Icon
```

Single infallible lookup. Name = UI semantic role ("document", "search", "alert"). Mimetype = optional content refinement. Fallback chain: exact name+mimetype → name+category → base name → universal fallback.

## Types

- `Icon` — name, label (a11y), paths, viewbox (24.0), stroke_width (2.0)
- `IconPath` — commands (&[u8] of native path cmds), layer (Primary/Secondary)
- `Layer` — Primary (main shape) or Secondary (detail, reduced opacity)

## Build Pipeline

```text
assets/icons/*.svg → build_icons.rs (in build.rs) → libraries/icons/data.rs
```

The manifest of curated icons (OS name, Tabler source, layer assignments) lives in `build_icons.rs`.

## Dependencies

- None (zero-dependency library)

## Adding Icons

1. Download the SVG to `assets/icons/`
2. Add an `IconEntry` to the `MANIFEST` in `build_icons.rs`
3. Rebuild — `data.rs` is regenerated automatically
