## Phase 1: Region Spec + Per-Process Endpoints (DONE)

The ASLR fix. No userspace changes.

1. Define region specs in `aslr.rs`: `(usable_size, entropy_bits)` for heap, DMA, device, stack
2. Compute outer region bounds from specs
3. Add `heap_end`, `dma_end`, `device_end` to `AslrLayout`
4. Add `heap_va_end`, `device_va_end` to `AddressSpace`, init from layout
5. `map_heap` checks `self.heap_va_end` instead of `paging::HEAP_END`
6. `map_device_mmio` checks `self.device_va_end` instead of `paging::DEVICE_MMIO_END`
7. `sys_memory_free` uses outer bound (fast reject), `unmap_heap` uses per-process
8. Update `paging.rs` constants for new outer bounds
9. `heap_va_size_is_constant_across_seeds` passes. All existing ASLR tests updated.
10. All kernel tests pass. All visual tests pass.

**Files:** `kernel/aslr.rs`, `kernel/address_space.rs`, `kernel/paging.rs`, `kernel/syscall.rs`, `kernel/system_config.rs`, `kernel/host/tests/kernel_aslr.rs`

## Phase 2: Heap VA Reclamation (DONE)

Independent of layout changes. Can parallel with Phase 1.

1. Add sorted free list of `(va, page_count)` to `AddressSpace`
2. `unmap_heap` adds freed VA to free list, coalesces adjacent
3. `map_heap` searches free list first (best-fit), bumps only if no fit
4. Tests: alloc/free/realloc cycles that exhaust bump-only VA, verify they succeed

**Files:** `kernel/address_space.rs`, new or existing heap test file

## Phase 3: Bootstrap Page (DONE)

Kernel-userspace contract. Enables Phases 4 and 5.

1. Define bootstrap page format at well-known VA (e.g., page zero, read-only)
2. Kernel maps bootstrap page during `process_create`
3. `sys` crate reads bootstrap page at init to discover region bases
4. `protocol::channel_shm_va()` reads from bootstrap page
5. Remove hardcoded `CHANNEL_SHM_BASE`, `SHARED_MEMORY_BASE`, `USER_STACK_TOP` from `system_config.rs`
6. Linker script no longer hardcodes stack top — kernel sets SP from layout

**Files:** `kernel/process.rs`, `kernel/system_config.rs`, `libraries/sys/`, `libraries/protocol/lib.rs`, `libraries/link.ld`, service `main.rs` files

## Phase 4: Handle-Mediated Channels

Replaces shared-memory channel IPC with kernel-mediated read/write. Processes
never map channel pages — the kernel copies 64-byte messages on their behalf.
Eliminates the channel SHM VA region entirely, simplifying the address space
layout and removing a class of cross-process memory exposure.

See `design/journal.md` entry "Handle-Mediated IPC" (2026-04-04) for rationale.

### Kernel additions

1. Port ring buffer read/write logic into kernel (~100 lines from `libraries/ipc/`)
2. Add `channel_read(handle, buf_ptr)` syscall — non-blocking, returns WouldBlock if ring empty
3. Add `channel_write(handle, buf_ptr)` syscall — writes one 64-byte message, returns WouldBlock if ring full
4. Kernel accesses channel shared pages via kernel VA (TTBR1) — no userspace mapping needed

### Kernel removals

5. Remove `map_channel_page` / `unmap_channel_page` from `AddressSpace`
6. Remove `next_channel_shm_va` bump allocator from `AddressSpace`
7. Remove channel SHM page mapping from `sys_channel_create` (lines 326-356)
8. Remove channel SHM page mapping from `sys_handle_send` (lines 661-701)
9. Remove `CHANNEL_SHM_BASE/END` constants and ASLR region spec for channel SHM
10. Remove `channel_shm_base` field from `BootstrapLayout` (72 -> 64 bytes)

### Kernel modifications

11. `channel_create` simplifies: allocate pages + insert handles, no mapping
12. `handle_send` for channels simplifies: transfer handle, no page mapping or rollback
13. `channel_signal` stays as explicit wake (preserves batched-write-then-signal pattern)

### Library changes

14. `sys`: add `channel_read(handle, buf) -> Result` and `channel_write(handle, buf) -> Result` wrappers
15. `ipc`: rewrite `Channel` to wrap syscalls — `from_base()` becomes `from_handle(handle, signal_handle)`. `send()` calls `sys::channel_write`. `try_recv()` calls `sys::channel_read`. All `unsafe` construction eliminated.
16. `protocol`: remove `channel_shm_va()` and `channel_shm_base()`

### Service changes (~12 files, mechanical)

17. Replace every `unsafe { Channel::from_base(channel_shm_va(N), PAGE_SIZE, endpoint) }` with `Channel::from_handle(handle)`
18. Signal calls unchanged (`sys::channel_signal(handle)`)

### Tests

19. New kernel host tests: `channel_read`/`channel_write` rights checks, empty ring returns WouldBlock, full ring returns WouldBlock, cross-process message delivery
20. Update `ipc_channel_create_pte_leak` — no PTEs to leak; test becomes "channel_create doesn't map pages"
21. Update `kernel_handle_send_rollback` — no page mapping to roll back for channels
22. Update bootstrap layout tests — size 64, field offsets shift, no `channel_shm_base`
23. All kernel tests pass. Full boot + visual tests pass (end-to-end IPC verification).

**Files:** `kernel/channel.rs`, `kernel/syscall.rs`, `kernel/address_space.rs`, `kernel/aslr.rs`, `kernel/paging.rs`, `kernel/system_config.rs`, `kernel/process.rs`, `kernel/host/tests/`, `libraries/sys/`, `libraries/ipc/`, `libraries/protocol/lib.rs`, all service `main.rs` files

## Phase 5: Kernel Completeness — Core Primitives

Independent of VA layout work. These are missing fundamental primitives identified
by systematic comparison against Zircon, seL4, QNX, and Mach (audit 2026-04-04).
Can be implemented in any order. No userspace changes required (new syscalls only).

### 5a. Handle duplication

`handle_dup(handle, rights_mask) → new_handle`. Creates a second handle to the
same kernel object in the caller's handle table, with attenuated rights. Add
`DUPLICATE` right (bit 10) controlling whether duplication is allowed.

1. Add `Rights::DUPLICATE` (bit 10), update `Rights::ALL`
2. Add `handle_dup` syscall: validate handle + DUPLICATE right, insert new entry with attenuated rights
3. Tests: dup succeeds, dup without DUPLICATE right fails, attenuated dup, dup of each handle type

**Files:** `kernel/handle.rs`, `kernel/syscall.rs`, `kernel/host/tests/`

### 5b. Exit code

Modify `exit` (syscall 0) to accept x0 = exit code (i64). Store on Process.
Retrievable by supervisor after process exit notification.

1. Add `exit_code: i64` field to Process
2. `sys_exit` stores x0 in process before teardown
3. Add `process_get_exit_code(handle)` syscall (or return from wait when process handle signals)
4. Tests: exit with code, supervisor reads code after exit notification

**Files:** `kernel/process.rs`, `kernel/syscall.rs`, `kernel/host/tests/`

### 5c. Timer set/cancel

`timer_set(handle, deadline_ns, period_ns)` — set or reset a timer's deadline.
Period 0 = one-shot, nonzero = auto-repeating. `timer_cancel(handle)` — disarm
without destroying. Handle remains valid.

1. Add `set` and `cancel` methods to timer module
2. Add `timer_set` and `timer_cancel` syscalls with WRITE right check
3. `timer_create` becomes: create handle + optional initial `timer_set`
4. Tests: set/reset, cancel before fire, periodic timer, cancel periodic

**Files:** `kernel/timer.rs`, `kernel/syscall.rs`, `kernel/host/tests/`

### 5d. Exception forwarding

When a process faults, send a structured exception message to a designated
channel instead of killing the process. The supervisor decides: resume, kill,
or debug. Extension point already exists (main.rs:1143, DESIGN.md §16.0).

1. Add `exception_channel: Option<ChannelId>` to Process
2. Add `process_set_exception_handler(process_handle, channel_handle)` syscall
3. In `user_fault_handler`: if exception channel set, write exception message (fault type, ESR, FAR, ELR, registers) to channel, block faulting thread
4. Add `exception_resume(thread_handle)` syscall to resume a faulted thread
5. If no exception handler, or handler channel is closed: kill process (current behavior)
6. Tests: exception handler receives fault, resume after handling, no handler = kill, handler channel closed = kill

**Files:** `kernel/main.rs`, `kernel/process.rs`, `kernel/syscall.rs`, `kernel/thread.rs`, `kernel/host/tests/`

### 5e. Priority inheritance for futexes

When a high-priority thread blocks on a futex, the holder temporarily inherits
the blocked thread's priority. Prevents unbounded priority inversion.

1. Extend `futex_wait` with optional `owner_tid` argument (or infer from futex value convention)
2. Track futex ownership in kernel: `futex_owner: Option<ThreadId>` per futex address
3. On block: if waiter priority > owner priority, boost owner (modify EEVDF weight/deadline)
4. On wake: revert boosted priority to original
5. Handle transitive inheritance (A waits on B, B waits on C → C gets A's priority)
6. Tests: basic PI, transitive PI, priority reverts after wake, multiple waiters

**Files:** `kernel/futex.rs`, `kernel/scheduler.rs`, `kernel/thread.rs`, `kernel/syscall.rs`, `kernel/host/tests/`

### 5f. Process memory read/write

`process_read_memory(process_handle, addr, buf, len)` and
`process_write_memory(process_handle, addr, buf, len)`. Required for debuggers
and crash analysis. Requires READ/WRITE right on process handle.

1. Add syscalls that walk target process's page tables to translate VA → PA, then copy via kernel VA
2. Target process must be suspended (all threads stopped) — prevents TOCTOU races
3. Tests: read/write succeeds on suspended process, fails on running process, respects rights

**Files:** `kernel/syscall.rs`, `kernel/process.rs`, `kernel/address_space.rs`, `kernel/host/tests/`

### 5g. Thread write state

`thread_write_state(thread_handle, state_ptr)`. Counterpart to existing
`thread_read_state` (syscall 38). Thread must be suspended.

1. Add syscall that writes register state to a suspended thread's saved context
2. Requires WRITE right on thread handle
3. Tests: write state to suspended thread, resume executes from new state, fails on running thread

**Files:** `kernel/syscall.rs`, `kernel/thread.rs`, `kernel/host/tests/`

### 5h. System info

`system_info(info_type) → value`. Query system-level properties.

1. Info types: `TOTAL_MEMORY`, `AVAILABLE_MEMORY`, `CPU_COUNT`, `PAGE_SIZE`, `BOOT_TIME_NS`
2. No capability required — these are not secrets
3. Tests: returns sensible values, unknown info_type returns InvalidArgument

**Files:** `kernel/syscall.rs`, `kernel/host/tests/`

## Phase 6: Channel Call + Batched Operations

Requires Phase 4 (handle-mediated channels). Builds on the channel_read/write
infrastructure.

### 6a. Channel call (synchronous RPC)

`channel_call(handle, send_buf, recv_buf, timeout_ns) → result`. Atomically:
write send message, signal peer, block until reply arrives on same channel.
One syscall instead of four. The idiomatic way to do RPC.

1. Kernel: write to send ring, set pending signal on peer, block caller on recv ring
2. On wake: read one message from recv ring into recv_buf, return to caller
3. Timeout: if deadline expires before reply, return WouldBlock (caller not stuck forever)
4. Tests: basic RPC round-trip, timeout, peer dies mid-call, concurrent calls on different channels

**Files:** `kernel/syscall.rs`, `kernel/channel.rs`, `kernel/host/tests/`

### 6b. Batched channel read/write

`channel_read_many(handle, buf, max_count) → count` and
`channel_write_many(handle, buf, count) → written`. One syscall for N messages.

1. Loop over ring buffer in kernel, copy up to N messages per syscall
2. `channel_write_many` does NOT auto-signal — caller signals explicitly after batch
3. Tests: batch read/write, partial batch (ring has fewer than max), empty/full ring

**Files:** `kernel/syscall.rs`, `kernel/channel.rs`, `kernel/host/tests/`

## Phase 8: PIE for Userspace

Requires Phase 3.

1. Add `code_base/code_end` to `AslrLayout` with 10-bit entropy
2. ELF loader maps at `code_base + segment_vaddr`
3. Apply relocations at load time (ref: kernel's `relocate.rs`)
4. Services compiled with position-independent flags
5. `SERVICE_PACK_BASE` becomes offset within code region
6. Update `build.rs` and linker script

**Files:** `kernel/process.rs`, `kernel/aslr.rs`, `build.rs`, `libraries/link.ld`

## Phase 9: T0SZ + Guard Pages

After Phase 1 (can follow any phase).

1. Change T0SZ from 28 (64 GiB) to 33 (8 GiB) in `boot.S` and `paging.rs`
2. Verify page table walk depth
3. Guard gaps are naturally unmapped — verify clean faults
4. Update `USER_VA_END`

**Files:** `kernel/boot.S`, `kernel/paging.rs`, `kernel/system_config.rs`

## Risks

- **HIGH:** Phase 4 changes the IPC data path for every service. Atomic cutover — all services update together. Mitigate with end-to-end boot test after kernel changes.
- **HIGH:** Phase 5e (priority inheritance) is subtle concurrency code in the scheduler hot path. Mitigate with property-based scheduler tests and stress testing.
- **HIGH:** Phase 8 requires build system changes for PIE. Use kernel's `relocate.rs` as reference.
- **MEDIUM:** Phase 4 moves ring buffer logic into kernel — bugs here are more dangerous than in userspace. Mitigate with thorough host tests (empty/full/rights/cross-process).
- **MEDIUM:** Phase 5d (exception forwarding) changes the fault handler, which is safety-critical. Mitigate with tests for every exception class.
- **MEDIUM:** Phase 2 adds allocator complexity. Mitigate with property-based tests.
- **LOW:** Phase 9 T0SZ change — standard AArch64 config, smoke test catches issues.
- **LOW:** Phases 5a-c, 5f-h are small, additive, low-risk syscalls.
